# 데이터 스키마 가이드

이 폴더의 JSON 3개가 대시보드의 "자동연동 저장 공간"입니다.
파일 내용을 바꾸고 새로고침하면 화면에 즉시 반영됩니다 (코드 수정 불필요).

공통 규칙
- `"_example": true` 가 붙은 항목은 화면 구조 확인용 예시입니다. 실제 데이터를 넣을 때 가장 먼저 지우세요.
- id는 항목마다 고유해야 합니다 (중복 시 마지막 항목으로 덮어써짐).
- 모르는 값은 빈 문자열 `""` 또는 `null`로 두세요. 화면에서는 "미입력/미확인"으로 표시됩니다. 값을 지어내서 채우지 마세요.

## 1. public-datasets.json — 공공데이터 카탈로그

| 필드 | 설명 | 예시 |
|---|---|---|
| category | A~E 중 하나 (meta.categories 참고) | "A" |
| title | 데이터셋 이름 | "농림축산식품부_도매시장 정산 가격 정보" |
| agency | 제공 기관 | "농림축산식품부" |
| format | 제공 형식 배열 | ["OpenAPI", "JSON", "XML"] |
| tags | 내부 분류 태그 (논문/시스템 매칭용, 자유롭게 정의) | ["§8-1"] |
| verified | 직접 접속·확인했는지 여부 | true / false |
| verified_date | 확인한 날짜 (YYYY-MM-DD) | "2026-06-10" |
| update_frequency | 갱신 주기 | "실시간" / "월별" / "부정기" |
| usage_count | 활용 건수 (조회수 등, 출처 페이지에 표기된 값) | 339 |
| description | 1~2문장 설명 | |
| url | 원본 링크 | |

## 2. income-data.json — 작목별 소득자료 (농사로)

10a(=300평) 기준 값으로 통일하는 것을 권장합니다 (농사로 원자료 단위와 동일).

| 필드 | 설명 |
|---|---|
| name | 작목명 |
| category | 분류 (노지과수/시설채소/식량작물 등 자유 정의) |
| total_revenue_per_10a | 총수입 (원/10a) |
| management_cost_per_10a | 경영비 (원/10a) |
| income_per_10a | 소득 = 총수입 - 경영비 (원/10a) |
| income_rate_pct | 소득률(%) = 소득/총수입×100 |
| unit_price_per_kg | 평균 단가 (원/kg) |
| yield_per_10a_kg | 생산량 (kg/10a) |

"내 농가 경영진단" 탭은 사용자가 입력한 면적·생산량·단가·경영비를 10a 단위로 환산해
이 배열의 같은 작목 항목과 비교합니다. 작목명이 정확히 일치해야 자동 비교가 됩니다.

## 3. rda-cases.json — 판매경로 진단용 가상 농가 케이스

`form` 객체는 `농가판매경로_AI시스템.jsx`의 입력 필드와 1:1 대응합니다.
`behaviorAnswers`의 q1~q5는 행동기반 성향진단 5문항 각각의 선택지 인덱스(0=A, 1=B, 2=C, 3=D)입니다.
케이스를 추가할 때는 두 객체를 모두 채워야 사례 선택 시 폼과 성향진단이 함께 자동 입력됩니다.
